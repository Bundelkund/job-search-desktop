# Examples

Fully fictional applications used as examples for the apply-skill workflow.

## `acme-solutions-architect/`

Persona: **Alex Beispiel** — Solutions Architect with five years of AI infrastructure experience and a parallel workshop-facilitation track. Applying to **Acme GmbH**, a fictional Berlin scale-up.

Files included:

- `stellenanzeige.md` — fictional job posting (Phase 1 input)
- `job-fit-analysis.md` — Phase 2a output
- `anschreiben.md` — Phase 3c cover letter, with adversarial-review note
- `cv.md` — Phase 2c CV in Markdown

## What is **not** in here

- Real personal data — every name, number, and company is invented
- PDFs — in the Desktop variant, PDF export is handled outside the skill (copy Markdown to your preferred editor)

## Verifying the example

Open `anschreiben.md` and `cv.md` in any Markdown reader to see what the final in-conversation artifacts look like. The `job-fit-analysis.md` shows the Phase 2a scoring structure.
